# Architecture Decision Records

ADRs capture the significant architectural decisions made in Helios — why we made them, what we considered, and what we gave up.

If you're new to the codebase and wondering "why is it built this way?", start here.

## How to read an ADR

Each record has five sections: **Status**, **Context** (the problem), **Decision** (what we chose), **Consequences** (tradeoffs), and **Alternatives Considered** (what we rejected and why).
